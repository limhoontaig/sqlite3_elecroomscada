네, 작성해주신 코드는 방향성이 매우 좋습니다! 인덱스를 IF NOT EXISTS로 안전하게 생성하도록 구성하셨습니다.

하지만 데이터 중복 저장과 8시 22분 멈춤 현상을 완벽하게 해결하기 위해서 딱 두 가지 중요한 부분을 더 수정해주셔야 합니다.

지금 코드에서는 실시간 원본 데이터 테이블(raw_data)에는 기본키(PRIMARY KEY)를 걸어 중복을 막으셨지만, 실제 3개씩 중복 데이터가 쌓였던 평균 데이터 테이블(hourly_avg)에는 아직 기본키(PRIMARY KEY)가 누락되어 있습니다.

가장 완벽하고 안전한 init_db() 최종 수정 코드를 안내해 드립니다.

🛠️ 최종 추천 init_db() 코드
아래 코드로 복사해서 적용하시면 됩니다. hourly_avg에도 기본키를 추가하고, 조회 성능을 극대화하기 위해 인덱스를 날짜+시간 복합 인덱스로 개선했습니다.

Python
def init_db():
    conn = sqlite3.connect(DB_NAME)
    c = conn.cursor()
    cols = ", ".join([f'"{name}" REAL' for name in DATA_LABELS])
    
    # 1. 실시간 데이터 테이블 (날짜와 시간을 묶어서 PRIMARY KEY 지정)
    c.execute(f'CREATE TABLE IF NOT EXISTS raw_data (log_date DATE, log_time TIME, {cols}, PRIMARY KEY (log_date, log_time))')
    
    # 🌟 [중요 수정] 평균 데이터 테이블에도 중복 저장을 막기 위해 PRIMARY KEY를 지정합니다.
    c.execute(f'CREATE TABLE IF NOT EXISTS hourly_avg (log_date DATE, log_time TIME, {cols}, PRIMARY KEY (log_date, log_time))')
    
    # 3. 최고/최저 데이터 테이블
    c.execute(f'CREATE TABLE IF NOT EXISTS daily_extremes (log_date DATE, extreme_type TEXT, {cols}, PRIMARY KEY (log_date, extreme_type))')
    
    # 🌟 [인덱스 개선] 날짜(log_date)와 시간(log_time)을 함께 인덱스로 잡으면 조회 속도가 갈라집니다.
    # 엑셀 보고서 출력이나 그래프 그릴 때 검색 속도가 수십 배 빨라집니다.
    c.execute('CREATE INDEX IF NOT EXISTS idx_raw_data_date_time ON raw_data (log_date, log_time);')
    c.execute('CREATE INDEX IF NOT EXISTS idx_hourly_avg_date_time ON hourly_avg (log_date, log_time);')

    create_field_inspection_table()
    conn.commit()
    conn.close()
⚠️ 기존에 이미 생성된 DB 파일(plc_logging_real.db) 주의사항
SQLite3의 특성상, 이미 테이블이 만들어져 있는 상태라면 코드에 PRIMARY KEY를 추가하더라도 CREATE TABLE IF NOT EXISTS 구문 때문에 기존 테이블 구조가 변경되지 않고 그냥 무시됩니다.

따라서 위 코드가 완전히 적용되어 중복을 원천 차단하려면 아래 방법 중 하나를 선택하셔야 합니다.

방법 A: 가장 간단함 (기존 DB 백업 후 새로 시작)
현재 테스트 중이거나 기존 데이터를 지워도 상관없으시다면, 프로그램을 끄고 AppData/Local/ElecRoomSCADA/plc_logging_real.db 파일을 지우거나 이름을 변경(plc_logging_real_backup.db)한 뒤 프로그램을 다시 켜시면 됩니다. 그러면 위 코드가 실행되면서 깨끗하고 완벽한 구조의 새 DB 파일이 생성됩니다.

방법 B: 기존 데이터 유지 (DB 관리 도구 사용)
만약 기존 데이터를 살려야 한다면, 프로그램이 꺼진 상태에서 DB Browser for SQLite 같은 프로그램으로 DB 파일을 열어 아래 쿼리를 직접 한 번 실행해 주시면 인덱스가 정상적으로 추가됩니다.

SQL
CREATE INDEX IF NOT EXISTS idx_raw_data_date_time ON raw_data (log_date, log_time);
CREATE INDEX IF NOT EXISTS idx_hourly_avg_date_time ON hourly_avg (log_date, log_time);
요약
올려주신 코드에서 hourly_avg 테이블에도 PRIMARY KEY (log_date, log_time)를 꼭 추가해 주세요. (중복 방지의 핵심)

인덱스는 날짜만 잡는 것보다 날짜와 시간을 묶어서 복합 인덱스로 잡는 것이 엑셀 출력 속도 향상에 훨씬 유리합니다.
